# Version Control Workflow Using Git
A beginner-friendly guide that explains the standard Git workflow for updating a project, including adding, committing, and pushing changes to GitHub. It provides step-by-step instructions and a simple daily routine to help maintain version control and ensure consistent progress in development.

## Main workflow:
git add .
git commit -m "your message"
git push

### Step 1: Save files
    in VS Code: ctrl + s

### Step 2: Check changes (optional but good habit)
    git status
It shows modified files and new files.

### Step 3: Add changes
    git add .
is used to stage all changes in the current directory and its subdirectories, preparing them to be included in the next commit.

### Step 4: Commit (save version)
    git commit -m "message"
 is described as a saved change. A commit message explains what change you made to your project.

### Step 5: Push (upload)
    git push
 is used to upload committed changes from your local repository to a remote repository 

## Daily Routine
Every time i finish coding:
    git add .
    git commit -m "what you did"
    git push
